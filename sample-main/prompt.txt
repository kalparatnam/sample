AMBIENT BACKGROUND SYSTEM (LIVING CANVAS)

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

This is the layer most "safe" sites skip — and it's the #1 reason

2024-2026 award-winning sites feel alive before any user input.

The background is never static. Choose ONE primary technique and

commit, then layer 1-2 supporting effects on top:

PRIMARY TECHNIQUES — pick based on aesthetic direction chosen above

Animated gradient mesh: 3-5 color blobs drifting on independent
sine-wave paths, heavy blur (60-120px), screen/overlay blend mode

→ best for: Liquid Organic, Dark Holographic, Glassmorphism 2.0

WebGL shader background (GLSL via OGL/Three.js): flowing noise
  (Perlin/Simplex), liquid distortion, chromatic aberration drift,
fractal grain — rendered to a fixed full-viewport canvas behind content

→ best for: Dark Holographic, Computational Brutalism

Aurora/blob field: 2-4 large radial-gradient divs, CSS-animated
position + opacity on staggered durations (15-30s loops, ease-in-out,

alternate), mix-blend-mode: plus-lighter or screen

→ best for: any dark-mode aesthetic, low GPU cost, easiest to ship

Particle/dot field (tsParticles or custom canvas): low-density,
slow independent drift, subtle size pulsing, NOT reactive to scroll

  (that's a different layer — this one breathes on its own clock)
→ best for: Editorial Black & White, Ultra-Minimal Swiss (use sparingly,

monochrome only)

Procedural noise/grain overlay (animated, not static): film-grain
texture cycling through 4-8 offset frames at 8-12fps via steps()

easing, low opacity (3-6%), blend-mode: overlay

→ best for: Paper & Texture, layered on top of ANY other technique

above as a finishing pass

AMBIENT CURSOR INFLUENCE (decouple from element-level cursor effects)

Background gradient/blob position offsets toward cursor with heavy
lag (300-600ms ease, large radius of influence) — should feel like

gravity, not tracking

Spotlight/vignette layer: radial-gradient mask following cursor at
low opacity, revealing a slightly different shade of the background

beneath — common in Glassmorphism and Dark Holographic builds

This must be ADDITIVE to the ambient animation's own independent
motion, never a replacement for it — background should still move

even if the user never touches the mouse

TECHNICAL RULES SPECIFIC TO THIS LAYER

Must run on its own requestAnimationFrame loop or CSS animation,
fully decoupled from scroll position and main thread JS logic

Fixed/absolute positioned, z-index below all content, pointer-events: none
Pause/reduce animation when tab is backgrounded (Page Visibility API)
to avoid battery drain — this is non-negotiable for production

prefers-reduced-motion: replace with a single static gradient frame,
never just "slow it down"

On mobile: either reduce particle/blob count by 60-70%, drop WebGL
shader to a CSS gradient fallback, or disable entirely below a

performance budget — ambient backgrounds are the first thing to

degrade gracefully, never the layout

This is a below-content asset — it must never compete with text
contrast/readability; run a contrast check against the busiest

frame of the animation, not just the resting frame
